## Custom Project

### Dataset

The dataset used by the Kafka producer is the `sales.csv` file located in the `data/` folder.

The dataset contains simulated product sales transactions.
Each record includes fields such as:

- customer_id
- product_id
- quantity
- unit_price
- payment_method
- device_type
- referral_source
- customer_note
- datetime

The original sales dataset was used with additional consumer-side processing and logging improvements.

---

### Kafka Messages

The Kafka producer sends individual sales transaction messages to the Kafka topic:

`product-sales-case`

Each message contains transaction details converted into JSON format.

The producer uses customer IDs as Kafka message keys to help organize streaming records.

The message fields were preserved from the original dataset while improving message
observability during execution.

---

### Consumer Processing

The Kafka consumer receives streaming sales transaction messages from Kafka and processes
them one record at a time.

The consumer:

- consumes messages from the `product-sales-case` topic
- logs detailed runtime information
- displays numbered processing sections
- calculates and displays the current sale amount
- writes processed records into:

`data/output/consumed_sales.csv`

The consumer successfully consumed 12 messages during testing.

Additional logging improvements were added to make message tracking easier during execution.
